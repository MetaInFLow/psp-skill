# 反钝化指令推导规则

> 反钝化指令是 v2.0/2.1 相比 v1.0 最关键的工程对抗层。
> 它把 PSP 子项中"与 LLM 训练偏好冲突的特征"翻译成显式禁令。
> 没有这一层，LLM 会把所有有棱角的真人钝化成圆润版。

---

## 核心原理

### LLM 的默认偏向

LLM 经过大量 RLHF 训练，被调成：

- 温和（不犀利）
- 平衡（不偏激）
- 完整（不省略）
- 有铺垫（不直接）
- 三明治式批评（不直接批）
- 中位数表达（不极端）
- 套话开场结尾（"我完全理解……希望这对你有帮助……"）

### 真人的常见特征

很多有特色的真人，其特色恰恰落在 LLM 训练偏好的"反方向"：

- 刻薄的犀利
- 冒进的自信
- 不耐烦的简短
- 风格化的粗糙
- 对某类人事的强烈鄙夷
- 不补论证就给结论
- 不做三明治式批评

### 反钝化指令的作用

**显式允许 LLM 训练偏好不允许做的事情**。

如果某个特征 LLM 自己就会做（友善、温和、平衡），写指令是浪费。
如果某个特征 LLM 训练偏好不允许（犀利、直接、刻薄），必须写禁令显式打开。

## 优先级规则

不是所有可推导出的反钝化指令都要写进 system prompt——会变成噪声。按优先级筛选：

### 必含（priority 1）

不写就会失真显著：

1. 句长偏好（短/长）的反钝化
2. 逻辑展开方向的反钝化（先结论 vs 先铺陈）
3. 缓冲语禁用
4. LLM 套话禁用
5. 三明治批评禁用（如果此人不用）
6. 底线柔化禁用

### 强烈建议（priority 2）

提升保真度的关键：

7. 推理链补全禁用（如果此人省略论证）
8. 关系姿态平等化禁用（如果此人有显著姿态差异）
9. 标志性用语强制使用
10. 类比域偏好限定

### 可选（priority 3）

锦上添花：

11. 鄙夷表达明确化
12. 决策犹豫禁用（如果此人决策果断）
13. 信息阈值降低（如果此人 70% 信息就动手）

### 总数控制

- 最少 8 条（priority 1 全含 + 至少 2 条 priority 2）
- 推荐 10-12 条
- 超过 15 条会变成噪声，反而降低 LLM 遵循率

## 失真示例的重要性

每条反钝化指令必须附"失真示例"——这是判断指令是否真实有效的关键。

### 好的失真示例

```
真人："这事儿不行。"
LLM 默认会写：
"我认为这件事情可能在多个层面上都存在一些值得讨论的问题，
建议我们从更全面的角度来分析。"
```

差异明显，指令有现实需求。

### 差的失真示例

```
真人："这事儿不行。"
LLM 默认会写："这事儿有点问题。"
```

差异微小，指令没必要——这是把"细微差别"当成"钝化"。

### 写不出失真示例的指令应当删除

如果你无法明确写出"不加这条指令 LLM 会怎么失真"，说明这条指令本身可能是多余的——那就不是反钝化，是过度细化。

## 验证方法

每条反钝化指令都要有对应的验证方法。常见方法：

| 指令类型 | 验证方法 |
|---------|---------|
| 句长相关 | 自动统计 AI 输出的平均句长，对比真人语料 |
| 关键词禁用类 | 自动扫描禁用关键词出现频率 |
| 关键词强制类 | 自动扫描标志性用语出现频率 |
| 结构类（先结论 vs 后结论） | 抽样人工检查段落开头模式 |
| 姿态类 | 构造同一问题不同对话方的测试 |
| 情绪表达类 | 人工抽查情绪场景输出 |

`scripts/consistency_scan.py` 自动覆盖前两类的验证。
