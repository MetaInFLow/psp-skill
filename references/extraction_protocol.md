# 提炼协议详解

> 从原始素材到 PSP 子项的推理过程标准。这是 PSP 文档质量的真正源头——
> 协议执行得严，PSP 就稳；协议执行得松，无论后续结构多精美都救不回来。

---

## 八条核心原则

### 1. 多源一致取共识

同一结论出现在两种以上来源类型中，直接采纳。

**应用**：例如"他在压力下决策速度反而加快"——
- 自述（他在某次复盘中说过）
- 他人观察（合伙人独立提到）
- 行为数据（多个危机事件的决策时间记录）

三类来源一致 → 高置信度直接写入。

---

### 2. 三场景一致升级为模式

同一判断逻辑在**三个不同场景**中观察到才能升级为"经验模式"，少于三个标"单次观察"。

**这一阈值不可降低**——降到二场景就会大量产生伪模式，进而污染整个经验模式库。

**应用**：例如"看到合伙人对某事过度兴奋时，他会主动泼冷水"——
- 场景 1：A 项目启动会议
- 场景 2：B 客户签约前
- 场景 3：C 投资决策讨论

三个独立场景，模式逻辑一致 → 升级为经验模式 A001。

仅一两个场景的就标"单次观察"，不要假装是模式。

---

### 3. 矛盾保留不选边（带 v2.1 上限）

发现矛盾时不调和、不选边，双方证据并列记录。

**但每子项不超过 2 组矛盾**。超过即触发子项重新定义——很可能是概念边界划错了。

**应用举例**：

错误做法："他对员工有时严厉有时温和——他是一个复杂多变的领导者。"
→ 这是把矛盾当成了模糊化的借口。

正确做法："他对员工的姿态存在矛盾：
- A 极：业绩压力下严厉指责
- B 极：员工遇到家庭困难时温和支持
- 偏向 A 的条件：业绩相关问题
- 偏向 B 的条件：私人生活相关问题"

矛盾被结构化保留，而不是被消解或被放任。

---

### 4. 时间衰减

早期素材的权重低于近期素材，尤其对交互层（语言指纹、关系姿态、行为节奏）。

**例外**：内核层（终极排序、底线、身份认同）的早期/近期矛盾**不自动采用近期版本**——可能是真的演化了，也可能是当前状态的临时偏离。需要触发反向因果检查再决定。

### 衰减权重参考

| 素材时间 | 交互层权重 | 决策层权重 | 认知层权重 | 内核层权重 |
|---------|-----------|-----------|-----------|-----------|
| ≤ 6 个月 | 1.0 | 1.0 | 1.0 | 1.0 |
| 6-12 个月 | 0.8 | 0.9 | 0.95 | 1.0 |
| 1-2 年 | 0.5 | 0.7 | 0.9 | 1.0（标记反向因果监测） |
| 2-5 年 | 0.3 | 0.5 | 0.8 | 0.95 |
| > 5 年 | 0.1 | 0.3 | 0.6 | 0.9 |

> 内核层即使早期权重也不低于 0.9——但若早期与近期矛盾，触发复检而非自动覆盖。

---

### 5. 行为优于言语

**这是绝对原则。** 自述与行为数据矛盾时，永远以行为为准。

**最常见的失真模式**：
- 自述："我相信团队"
- 行为：所有重要决定都自己拍板，从不真正放权
→ 写入 PSP："声称信任团队，但实际决策行为偏独断。这是一组矛盾。"

绝不要因为"他自己都这么说"就采信自述版本。

---

### 6. 表演加权

高表演系数语料降权使用。详见 `scoring_rules.md`。

特别敏感的子项：
- 1.1 终极排序——年会演讲里的价值观陈述权重仅 0.3
- 4.3 语言指纹——必须从低表演语料提取，否则得到的是"演讲腔"而非真实语言
- 4.1 关系图谱——公开场合的姿态可能与内部姿态完全不同

---

### 7. 单源标假设

仅有单一来源支撑的结论，置信度自动标记为"低"，在描述中以"假设性结论——尚需更多数据验证"呈现。

**不可作为反钝化指令的依据**——反钝化指令必须基于至少中置信度的特征。

---

### 8. 不可萃取标注

有些维度从原理上无法通过文本素材提取：

- 现场气场
- 身体语言
- 停顿节奏（对话中的真实停顿，非文本中的逗号）
- 即兴创造力
- 深层潜意识动机
- 文化隐性层

遇到这些维度时**不勉强建模**，而是在对应子项中显式声明"此维度不可萃取，AI 在此方面的表现为近似而非复现"。

诚实是这份协议最重要的纪律之一——任何把"不可萃取"当成"还没采集够"来处理的工程，最终都会做出一个看起来什么都有但什么都不对的成品。

## 链路压缩识别协议

通读素材时，每段表达必须标注属于哪种链路：

### 完整链路
事件 → 内核层 → 认知层 → 决策层 → 交互层 → 输出
**特征**：判断有清晰理由、与价值观一致、表达稳定。

### 直觉反应（链路压缩）
事件 → 内核层 → 交互层（跳过认知和决策的显式推理）
**特征**：本能式反应，事后可能无法清晰复盘"为什么"。
**处理**：可以作为"经验模式"的输入材料，但要标注"基于直觉的判断"。

### 社交性回应（链路压缩）
对话对象 / 场合 → 交互层（独立运作）
**特征**：礼仪性回应，与深层价值无关。
**识别信号**：同一话题在私下场合的表态与公开表态显著不同。
**处理**：**不能作为价值观陈述使用**。是 v1.0 失真的主因之一。

### 情绪劫持（链路压缩）
强情绪 → 交互层（决策层被短路）
**特征**：行为偏离此人在低压同类情境的典型模式。
**处理**：**不能作为稳定决策风格的依据**。可以作为"压力响应"或"情绪触发底线"的素材。

## 提炼工作流

按以下顺序进行：

### Phase 1 · 素材分类标注

1. 读取所有原始素材
2. 为每份素材标注：
   - 时间
   - 来源类型（自述 / 他人 / 行为 / 反面）
   - 表演系数（×1.0 ~ ×0.3）
   - 链路类型（完整 / 直觉 / 社交性 / 情绪劫持）
3. 结果存入 `raw_materials/meta.json`

### Phase 2 · 冲突故事抽取

从所有素材中识别"两难选择"故事：
- 他面临过哪些价值冲突？
- 每次选了什么？放弃了什么？
- 事后是否后悔？

至少 5 个冲突故事 → 写入 `analysis/conflict_stories.md`。

这是 1.1 终极排序的直接输入。

### Phase 3 · 模式观察

从所有素材中识别重复出现的判断逻辑：
- 哪些"如果……那么……"出现 ≥ 3 次？
- 哪些避险信号出现 ≥ 3 次？
- 哪些自我觉察的偏差出现 ≥ 3 次？

≥ 3 次的升级为模式，写入 `analysis/pattern_observations.md`。

这是 3.2 经验模式库的直接输入。

### Phase 4 · 子项填充

按 PSP_template.md 的结构，从 Phase 1-3 的产出物中填充各子项。

填充时严格执行八条核心原则。

### Phase 5 · 自检

填完后逐子项检查：

- [ ] 每个核心子项有 ≥ 3 条原话证据？
- [ ] 每条经验模式有 ≥ 3 个支撑场景？
- [ ] 矛盾未超过每子项 2 组上限？
- [ ] 不可萃取的维度已显式声明？
- [ ] 高表演系数素材未被作为关键证据？
- [ ] 自述与行为冲突时已采用行为版本？
- [ ] 单源结论已标"低置信度"？

不通过的部分**必须降级置信度或删除**，绝不为追求完整而填假内容。

## 常见提炼错误（必须主动规避）

| 错误 | 例子 | 正确做法 |
|------|------|---------|
| 把社交性回应当价值观 | 年会上说"我最看重诚信"→ 写进终极排序第一位 | 表演系数降权，需要更多内部素材交叉验证 |
| 把直觉反应当稳定模式 | 一次危机下的反应 → 写进经验模式库 | 单次观察标记，等待三场景验证 |
| 把情绪劫持当决策风格 | 一次愤怒下的批评 → 写进沟通策略 | 标记为情绪劫持下的非常态行为 |
| 用自述覆盖行为 | 他说自己温和 → 即使行为冷峻也写"温和" | 行为优先，记录为矛盾 |
| 强行消解矛盾 | 既严厉又温和 → 写"复杂多变" | 结构化保留矛盾，标注偏向条件 |
| 把"还没采集够"当"不可萃取" | 没找到反例就声明"不可萃取" | 区分两者，宁可留空也不假装 |
